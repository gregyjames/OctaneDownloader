## 2025-05-14 — [ThrottleStream Async Optimization]
**Learning:** The `ThrottleStream` was using a "sync-over-async" anti-pattern where its asynchronous methods (`ReadAsync`) were calling a synchronous `Throttle` method that blocked the calling thread using `AutoResetEvent.WaitOne()`. This defeats the purpose of asynchronous I/O and can lead to thread pool starvation in highly concurrent scenarios like multi-part downloads.
**Action:** Always provide a truly asynchronous path (e.g., `ThrottleAsync`) using non-blocking delays like `Task.Delay` for asynchronous stream operations. Ensure all `ReadAsync` and `WriteAsync` overloads are overridden to use this path.
